# LCPR 代码生成提示词（精简版）

## 核心规范

### 1. 目录结构
```
py/{题目编号}.{题目名称}/
├── solution.py              # 主解法（最优方法）
├── solution2-{方法名}.py   # 解法2
├── solution3-{方法名}.py   # 解法3
└── README.md               # 题目说明
```

### 2. 代码格式

```python
# @lc app=leetcode.cn id={编号} lang=python3
# @lcpr version=
#
# [{编号}] {题目名称}
#

# @lc code=start
from typing import Optional, List

class Solution:
    def functionName(self, ...):
        """
        方法描述：{方法名称}
        时间复杂度：O(?)
        空间复杂度：O(?)
        
        思路：
        1. 关键点1
        2. 关键点2
        """
        # 实现代码
        pass
# @lc code=end

# @lcpr case=start
# {测试用例}
# @lcpr case=end
```

### 3. 多解法要求

- **至少3-4种解法**
- **方法1（solution.py）**：最优/最常见方法
- **方法2-4**：不同思路（迭代、递归、BFS、DFS等）
- **每个解法独立文件**，包含完整注释和测试用例

### 4. README格式

```markdown
# {编号}. {题目名称}

## 题目链接
https://leetcode.cn/problems/{slug}/

## 难度
Easy/Medium/Hard

## 解法

### 方法1：{名称}（推荐⭐）
- 文件：`solution.py`
- 时间复杂度：O(?)
- 空间复杂度：O(?)
- 思路：...

### 方法2：{名称}
- 文件：`solution2-{方法名}.py`
- 时间复杂度：O(?)
- 空间复杂度：O(?)
- 思路：...

## 复杂度对比表
| 方法 | 时间复杂度 | 空间复杂度 | 推荐度 |
|------|-----------|-----------|--------|
| 方法1 | O(?) | O(?) | ⭐⭐⭐⭐⭐ |
```

### 5. 必须包含

- ✅ 完整的函数注释（方法描述、复杂度、思路）
- ✅ 关键步骤的中文注释
- ✅ 至少3个测试用例
- ✅ 类型提示（Python）
- ✅ 边界情况处理
- ✅ README.md说明文档

### 6. 文件命名

- `solution.py` - 主解法
- `solution2-brute-force.py` - 暴力
- `solution2-two-pointer.py` - 双指针
- `solution2-dp.py` - 动态规划
- `solution2-iterative-dfs.py` - 迭代DFS
- `solution2-recursive.py` - 递归
